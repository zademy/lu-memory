# Prototype Pattern Implementation Skill

This skill guides Cascade through implementing the Prototype pattern to securely manage stateful beans or clone complex objects in Spring Boot.

## Concept & Analogy

**Analogy:** Cell division (Mitosis). The original cell acts as a prototype and actively participates in creating an exact copy of itself, rather than trying to build a new cell from scratch from the "outside."
**When to apply:** When your code shouldn't depend on the concrete classes of objects you need to copy, especially when dealing with objects passed via interfaces or when object initialization is extremely expensive.

## Prerequisites

1. Identify if the goal is to manage a stateful Spring Bean (needs a fresh instance per use) OR to clone pure Domain Objects/DTOs.
2. Verify isolation requirements: Is this resolving a thread-safety issue (Race Condition) in a Singleton?

## Implementation Steps

### Option A: Context-Level Prototype (Spring Bean Scope)

#### Step A1: Annotate the Target Component

Locate the class that holds state. It MUST NOT be a standard Singleton.

- Ensure the class is annotated with `@Component` or `@Service`.
- Add the scope annotation: `@Scope(ConfigurableBeanFactory.SCOPE_PROTOTYPE)`.

#### Step A2: Refactor the Injector (The Caller)

Locate the Singleton service that needs to use this prototype.

- **CRITICAL CHECK:** Do NOT inject the Prototype directly via `@Autowired PrototypeBean bean;`. This creates a Scoped Proxy trap (the prototype gets frozen inside the singleton).
- Instead, inject an ObjectProvider: `@Autowired ObjectProvider<PrototypeBean> provider;`.

#### Step A3: Retrieve Fresh Instances

Where the proxy is needed execute:

- `PrototypeBean freshInstance = provider.getObject();`
- Use the instance. A new clone is generated by Spring automatically.

### Option B: Data-Level Prototype (Deep Object Cloning)

#### Step B1: Discard Legacy Cloneable

If the codebase uses the legacy `Cloneable` interface or `Object.clone()`, remove them. They only perform shallow copies and are considered an anti-pattern.

#### Step B2: Use MapStruct (Preferred)

If cloning DTOs or Entities:

- Create a MapStruct `@Mapper` interface.
- Define a method `TargetClass clone(TargetClass source);`
- Use the generated mapper to create deep copies seamlessly.

#### Step B3: Use Copy Constructors (Alternative)

If MapStruct is not available:

- Create a constructor in the target class that accepts an instance of itself: `public TargetClass(TargetClass source) { ... }`
- Manually deep-copy all internal fields and nested objects.
