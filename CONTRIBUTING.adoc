== 🤝 Código de Conducta

Al participar en este proyecto, se espera que mantengas prácticas de respeto y colaboración propias del código abierto. Por favor, sé inclusivo, respetuoso y constructivo en todas tus interacciones.

== ⚙️ Configuración y Desarrollo

### Requisitos Previos

* *Java JDK 25* o superior.
* *Maven* (el proyecto incluye un wrapper de Maven `mvnw`, por lo que no es estrictamente necesario tenerlo instalado a nivel global).
* *Git*.

### Construir desde el Código Fuente

1. Clona el repositorio:
+
[source,bash]
----
git clone <url_del_repositorio>
cd lu-memory
----

2. Compila el proyecto utilizando el wrapper de Maven:
+
[source,bash]
----
./mvnw clean install
----
+
_(En Windows, utiliza `mvnw.cmd clean install`)_

3. Ejecuta la aplicación:
+
[source,bash]
----
./mvnw spring-boot:run
----

El proyecto utiliza **SQLite** como base de datos, la cual se creará automáticamente en el entorno local gracias a la configuración de JPA.

== 🐞 Reporte de Bugs

Si encuentras algún error o comportamiento inesperado, por favor crea un *Issue* en el repositorio. Asegúrate de incluir:

* Un título claro y descriptivo.
* Los pasos exactos para reproducir el problema.
* El comportamiento esperado vs. el comportamiento actual.
* Trazas de error (stack traces) o logs del servidor, si aplican (por favor, usa bloques de código para formatearlos).
* Detalles de tu entorno (Sistema Operativo, versión de Java/Spring).

== 💡 Sugerir Mejoras

¡Las nuevas ideas son siempre bienvenidas! Para proponer una nueva funcionalidad o mejora:

* Abre un *Issue* describiendo la mejora.
* Explica *por qué* esta mejora sería útil o cómo mejoraría la interacción de los agentes de IA con el servidor MCP.
* Si tienes una idea de cómo implementarlo a nivel técnico (ej. nuevos endpoints de MCP, cambios en entidades de base de datos), inclúyelo allí.

== 🛠️ Proceso de Pull Request (PR)

1. Haz un **Fork** del repositorio y crea tu rama a partir de `main`.
2. **Nomenclatura de ramas**: Usa un nombre descriptivo para tu rama, como `feature/nueva-herramienta-mcp` o `fix/error-busqueda-fts`.
3. Si estás agregando código nuevo que requiera pruebas, por favor añade los tests correspondientes.
4. Actualiza el `README.md` o cualquier otra documentación relevante si tus cambios afectan la interfaz pública o la configuración.
5. Asegúrate de que las pruebas automatizadas pasen exitosamente ejecutando:
+
[source,bash]
----
./mvnw test
----
6. ¡Abre tu Pull Request!

== 💻 Guías de Programación

* **Lenguaje y Framework**: Desarrollamos en *Java 25* y *Spring Boot 4.x*. Por favor, sigue las convenciones modernas de Java.
* **Arquitectura MCP**: El proyecto se centra en exponer herramientas (tools) para agentes LLM a través del Model Context Protocol utilizando `spring-ai-starter-mcp-server`. Cualquier nueva "herramienta" debe seguir esta filosofía y registrarse adecuadamente.
* **Base de datos**: Usamos SQLite integrado con JPA/Hibernate. Evita consultas SQL nativas específicas de un motor a menos que sea estrictamente necesario (ej. búsquedas FTS de SQLite), para mantener el código lo más limpio posible.
* **Compatibilidad**: Si modificas las entidades de las memorias (Observaciones, Sesiones, Prompts), asegúrate de tener en cuenta la retrocompatibilidad, ya que los usuarios tendrán bases de datos SQLite locales existentes.

== 📝 Mensajes de Commit

* Usa el tiempo presente en imperativo ("Añade funcionalidad" en vez de "Añadí funcionalidad" o "Añadida funcionalidad").
* Limita la primera línea a 72 caracteres o menos.
* Haz referencia a los números de Issues o PRs si es que aplican.
